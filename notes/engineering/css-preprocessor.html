# CSS Modules

把CSS视作一个独立的模块，在js中import使用

规范：https://github.com/css-modules/css-modules (opens new window)

基础用法：http://www.ruanyifeng.com/blog/2016/06/css_modules.html (opens new window)

VueCLi的集成使用：https://cli.vuejs.org/zh/guide/css.html#css-modules (opens new window)

<template>
  <div>
    <p :class="{ [$style.red]: isRed }">
      Am I red?
    </p>
    <p :class="[$style.red, $style.bold]">
      Red and bold
    </p>
  </div>
</template>

<script>
export default {
  created () {
    console.log(this.$style.red)
  }
}
</script>

<style module>
.red {
  color: red;
}
.bold {
  font-weight: bold;
}
</style>

# SCSS Lint

早期的 scss-lint (opens new window) 支持的是已经被淘汰的 Ruby Sass；

使用方案：

  • stylelint (opens new window)
    • postcss (opens new window)
    • ...postcss plugins
  • stylelint-config-recommended-scss (opens new window)
    • stylelint-config-recommended (opens new window)
    • postcss-scss (opens new window) ( Parse SCSS syntax and apply PostCSS transformations directly to SCSS source code. )
    • stylelint-scss (opens new window)
  • stylelint-config-recess-order (opens new window)
    • stylelint-order (opens new window)
  • stylelint-prettier (opens new window)
  • stylelint-config-prettier (opens new window)
npm i -D stylelint stylelint-config-recommended-scss
npm i -D stylelint-config-recess-order
npm i -D stylelint-prettier stylelint-config-prettier

安装vscode插件 stylelint (stylelint-plus supports auto fix on save)，然后修改配置，添加对 scss 文件的校验

项目配置文件，rules中添加规则覆盖

/** .stylelintrc.js **/
module.exports = {
  extends: ['stylelint-config-recommended-scss', 'stylelint-config-recess-order', 'stylelint-prettier/recommended'],
  rules: {
    'selector-pseudo-class-no-unknown': [true, { ignorePseudoClasses: ['global', 'export'] }]
  }
}

/** .stylelintignore **/
dist/*
public/*

命令行检测和修复，可以配合 .stylelintignore 使用

npx stylelint "**/*.scss"
npx stylelint --fix "**/*.scss"

disable complains：https://stylelint.io/user-guide/ignore-code/ (opens new window)

/*stylelint-disable*/

# less-vs-sass-vs-stylus

npm trends (opens new window)

  • 变量、嵌套、模块、mixins、继承、运算符、内建函数、控制流

  • Less和Stylus都是用JavaScript写的，能直接用在浏览器端和node，Sass用Dart写的，需要在服务端做处理， 但npm发布的package是纯JavaScript的

  • Less不能使用条件语句、不能自定义function、不能使用带参数的mixins，SCSS的extend也更直观好用

  • stylus差异较大，采用缩进，其它特性的语法也偏简洁

  • 总之，SCSS更面向编程，less只能说是css的拓展，不喜欢Stylus的风格，趋势热度上亦或是语法功能上，都倾向于选择scss
